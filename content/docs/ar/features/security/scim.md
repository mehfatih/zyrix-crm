# توفير SCIM 2.0

**ما يفعله:** مزامنة المستخدمين والمجموعات تلقائياً بين مزود الهوية (Okta، Azure AD، Google Workspace، JumpCloud) وزيركس. عندما تضيف الموارد البشرية موظفاً، يظهر في زيركس تلقائياً. عندما يغادر شخص، يُبطل الوصول فوراً.

## مثال من الواقع

**مثال حقيقي: "بنك سعودي ٥٠٠ موظف"**

بنك متوسط الحجم ينتهك سياسته الأمنية: الموظفون السابقون كان لا يزال لديهم حسابات CRM نشطة أشهراً بعد المغادرة، لأن IT أبطل يدوياً عبر ٢٠+ نظام. تدقيق SOC 2 أشار لهذا كاكتشاف حرج.

تكامل SCIM لزيركس مع نظام هوية Okta لديهم: إنهاء الموظف في الموارد البشرية → إبطال وصول زيركس خلال ٦٠ ثانية. **تم إغلاق اكتشاف SCOC 2. تم القضاء على مخاطر المطلعين.**

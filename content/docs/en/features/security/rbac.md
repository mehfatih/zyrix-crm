# Role-Based Access Control (RBAC)

**What it does:** Create custom roles with precise permissions. Example: "Sales Viewer" can see customer data and revenue but can't edit or delete. "Finance Admin" can see all financial data but not customer communications. 25 granular permissions across modules.

## Key Benefits

- **Principle of least privilege:** People see only what they need
- **Separation of duties:** Compliance requirement for regulated industries
- **Reduced insider risk:** Disgruntled employees can't export everything
- **Contractor-friendly:** Give temporary access to specific modules

## Real-World Example

**Real-World Example: "Jeddah Accounting Firm"**

A 40-person accounting firm had the problem of junior accountants accessing senior partners' client revenue data. One junior left to a competitor with a list of high-value clients.

With RBAC, juniors got custom roles that hide revenue columns entirely. Only partners saw revenue. **Security incident eliminated. Compliance with ISCA/SOCPA data protection standards achieved.**
